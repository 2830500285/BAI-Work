# 第 16 页本地化用户价值设计

## 沟通任务

答辩评委在看完第 16 页后，应理解 BAI Work 的本地化优势不是增加一个聊天入口，而是降低普通用户使用优质模型完成真实任务的门槛。

## 方案比较

1. 新增独立优势页。信息最突出，但会增加页数并拉长既定的 12 分钟答辩节奏。
2. 重写现有第 16 页。沿用三卡布局，把目标用户页升级为本地化用户价值页，不改变 19 页结构。
3. 仅在现有页脚增加一句。改动最小，但不足以形成评委可记忆的产品判断。

## 决策

采用方案 2。三张卡分别表达：

- 从聊天到交付：不使用“国内用户大部分”等未经验证的比例判断，改为描述可观察的使用摩擦。
- 统一模型入口：通过 BAI 默认服务或用户配置的兼容服务使用模型能力；用户必须提供已获授权的 API Key。
- 普通用户可完成：突出配置极简、过程可见和产物可验收，说明该价值对非开发者尤其重要。

## 表述边界

- 可点名 Claude、GPT，因为当前 BAI provider preset 已包含相应模型 ID。
- 不宣称与 OpenAI 或 Anthropic 存在官方合作、背书或直接集成关系。
- 不暗示绕过地区限制；模型服务以用户授权、实际可用性和相关服务条款为准。
- 不宣称已有量化市场验证；页脚继续明确这是基于 Jobs-to-be-Done 的产品判断。

## 验收标准

- 第 16 页仍使用现有三卡模板，标题、正文和底部结论没有异常换行或溢出。
- 19 页顺序、页码、模板 chrome、页脚和讲稿备注保持完整。
- 可复现源文件、PPTX 和讲稿同步更新。
